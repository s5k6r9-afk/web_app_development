# 產品需求文件 (PRD) - 主題占卜系統

## 1. 專案概述
- **背景與動機**：現代人生活壓力大，常在感情、工作、人際等方面遇到迷惘與不確定性。本系統旨在提供一個簡單、直覺的線上占卜平台，讓使用者能透過選擇不同的主題（如愛情、事業、財運）進行抽牌或抽籤，獲得心理上的指引與娛樂體驗。
- **目標用戶**：對未來感到迷惘、需要簡單建議或指引、或單純對星座、塔羅、占卜感興趣的一般大眾。
- **核心價值主張**：提供快速且具備神祕感的占卜體驗，幫助使用者在猶豫不決時獲得不同視角的啟發與療癒，並提供簡單易懂的解讀結果。

## 2. 功能需求

1. **多重主題占卜選擇**
   - 作為 **一般用戶**，我希望 **可以在首頁選擇不同的占卜主題（如愛情、事業、財運、健康）**，以便 **針對我目前的煩惱進行專屬的占卜**。

2. **虛擬抽牌 / 抽籤機制**
   - 作為 **一般用戶**，我希望 **能有互動式的抽牌或抽籤動畫與機制**，以便 **在線上也能感受到真實占卜的儀式感。**

3. **占卜結果解讀與展示**
   - 作為 **一般用戶**，我希望 **在抽完牌後能立刻看到詳細的牌義或籤詩解讀**，以便 **獲得具體且具啟發性的建議。**

4. **會員註冊與登入系統**
   - 作為 **註冊用戶**，我希望 **能夠建立並登入個人帳號**，以便 **系統能記住我的身分並使用進階功能。**

5. **歷史占卜紀錄保存**
   - 作為 **註冊用戶**，我希望 **系統能自動保存我過去的占卜結果與日期**，以便 **我可以在未來回顧並驗證占卜的準確度或反思當下的狀態。**

6. **每日一抽（當日運勢）**
   - 作為 **一般/註冊用戶**，我希望 **每天登入時能進行一次免費的「每日一抽」**，以便 **獲得當日運勢的小提醒或正向鼓勵。**

## 3. 非功能需求
- **技術限制**：
  - 網頁後端採用 **Flask** 框架。
  - 前端樣式與呈現採用 **Jinja2** 模板引擎進行渲染。
  - 資料庫採用 **SQLite**，以輕量化方式儲存使用者帳號與歷史紀錄。
- **效能與安全考量**：
  - 密碼需經過加密（如 bcrypt）後儲存，確保會員資料安全。
  - 圖片（如塔羅牌圖）需進行壓縮與快取，確保頁面載入速度流暢，提升使用者體驗。
  - 占卜邏輯在後端進行，防止使用者透過竄改前端代碼作弊取得特定結果。

## 4. MVP 範圍 (最小可行性產品)

- **Must Have (必須有 - MVP 核心)**
  - 基本的版面佈局與首頁（Jinja2 模板）。
  - 至少 3 種主題分支供選擇（愛情 / 事業 / 綜合）。
  - 隨機抽結果的演算法。
  - 結果展示頁面（文案與圖案）。
  
- **Should Have (應該有)**
  - 會員註冊、登入/登出機制。
  - 提供會員專屬的「歷史紀錄」查詢頁面（SQLite 寫入/讀取）。
  - 抽牌時提供簡單的前端互動或過場效果。

- **Nice to Have (加分項)**
  - 每日運勢機制（限制 24 小時內僅能抽一次）。
  - 占卜結果分享功能（一鍵複製連結或圖文）。
  - 後台管理系統：允許管理員新增或修改塔羅牌/籤詩的解讀文案。
